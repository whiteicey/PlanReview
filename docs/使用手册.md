# 开发方案审查助手 · 使用手册

> 适用对象：油气田开发方案的编制、校核、审查相关人员。
> **无需任何编程或计算机专业知识**，按本手册一步步操作即可。

---

## 一、一句话说明：这个工具能帮你做什么

你把一份 **Word 格式（.docx）的开发方案** 交给它，它会在几秒钟内自动帮你**初步检查**方案里常见的低级错误和前后矛盾，比如：

- 井数前后对不上（摘要写 40 口、汇总表算出来 38 口）；
- 高峰产量超过了地面处理能力；
- 关键参数换了数值却没写调整原因；
- 审查意见回复表里漏填了处理状态；
- 同一个东西在正文里一会儿叫「开发井总数」一会儿叫「部署井数」，没统一。

它会把发现的每个问题列成一张张**问题卡片**，告诉你问题在哪、属于哪一类、有多严重、建议怎么办。

### ⚠️ 必须先记住的一件事

**这是「AI 初审」，不是「正式审查结论」。**

- 它只是帮你**提前发现明显问题**，节省人工反复核对的时间；
- 它**不能代替**专家审查，也**不能作为正式审查依据**；
- 它说「没发现问题」**不等于**方案就正确；
- 所有结果**都必须由有资质的专家复核后**才能采信。

页面上、每一份导出的报告里，都会一直印着这句话：**「AI 初审结果，不是正式审查结论」**。请始终这样理解它。

## 二、开始之前：让它「跑起来」

这个工具运行在你自己的电脑上（或单位内部的一台电脑上），**不联网、不上传你的文件到外部**。

### 2.1 你需要谁帮忙

第一次使用前，需要**信息科/技术同事**帮你做一次性的启动准备（他们照着随附的《技术说明 README》做即可，大约几分钟）。你只需要拿到两样东西：

1. 一个**网址**，形如：`http://127.0.0.1:8765`
2. 确认技术同事已经把服务「打开」了（这台电脑上的程序正在运行）。

> 小知识：`127.0.0.1` 的意思是「就是这台电脑自己」。所以这个网址**只能在装了程序的那台电脑上打开**，别人从别的电脑访问不了——这是为了安全，保证方案资料不外泄。

### 2.2 确认「已就绪」

1. 打开电脑上的浏览器（Chrome、Edge 都行）。
2. 在地址栏输入上面的网址 `http://127.0.0.1:8765`，回车。
3. 如果看到页面上出现大标题 **「开发方案审查助手」**，说明已经就绪，可以开始用了。
4. 如果页面打不开、提示「无法访问」，说明服务没打开——请联系技术同事帮你启动。

## 三、一步步操作（图文对照）

整个过程只有三步：**选文件 → 点按钮 → 看结果**。

### 第 1 步：打开页面

浏览器打开 `http://127.0.0.1:8765`，你会看到：

- 顶部大标题「**开发方案审查助手**」；
- 下面一行浅色小字：「用可追溯的证据链完成文本型 DOCX 形式初审」；
- 一个白底方框，写着醒目的一句：「**AI 初审结果，不是正式审查结论**」；
- 一个写着「**选择 DOCX**」的按钮；
- 旁边一个橙色的「**上传并开始**」按钮。

### 第 2 步：选择你的方案文件

1. 点击「**选择 DOCX**」按钮。
2. 电脑会弹出一个「打开文件」的窗口，让你从电脑里挑文件。
3. 找到你要检查的开发方案，**它必须是 `.docx` 结尾的 Word 文档**（下面「常见问题」会讲为什么）。
4. 选中它，点「打开」。

### 第 3 步：开始初审

1. 点橙色的「**上传并开始**」按钮。
2. 页面会先显示「正在上传并建立本地案例…」，再显示「正在执行初审，请稍候…」。
3. 等几秒钟（一般 3～10 秒），结果就会出现在下方。

## 四、怎么读结果

初审完成后，结果区从上到下依次是：

### 4.1 顶部概要

- 「**已应用 12 条审查规则**」——表示规则库已加载，做了完整检查。
  - 如果这里显示的是一条**黄色提示条**「未加载规则库，本次仅做 AI 复核……」，说明规则库没配置好，本次只做了很浅的检查，请联系管理员（见「常见问题」）。
- 一行统计：**案例编号**、**发现问题 N 条**、**提取参数 N 个**。案例编号是这次检查的唯一编号，导出报告、后续查询都用它。

### 4.2 问题卡片（重点）

每发现一个问题，就是一张卡片。一张卡片包含：

| 卡片上的内容 | 意思 |
|---|---|
| **左上角彩色标签**（如「汇总核对」「跨专业」） | 问题的**类别**（见 4.3） |
| **严重度：高 / 中 / 低** | 问题的严重程度，高的优先看 |
| **来源：规则 / AI 复核** | 「规则」= 按固定规则算出来的；「AI 复核」= AI 模型提示的，需更谨慎看待 |
| **需人工复核**（灰色小标签） | 表示这条**必须由人来判断**，工具不敢下结论 |
| **加粗标题**（如「开发井总数需复核」） | 问题的一句话概括 |
| **相关参数** | 涉及哪个参数（如「高峰产量」） |
| **说明** | 问题具体是什么（如「求和不一致」「超过处理能力」） |
| **建议** | 建议你怎么做（一般是「请补充证据并由专家复核」） |
| **原文证据 N 处** | 这个问题在原文里有 N 个位置作依据；**具体在第几段/第几张表，需导出报告查看**（见第六节） |

### 4.3 类别是什么意思

| 类别 | 含义 | 举例 |
|---|---|---|
| **一致性** | 同一个参数在不同地方数值应一致 | 开发井总数在摘要和正文不一致 |
| **汇总核对** | 分项之和应等于合计 | 生产井数 + 评价探井数 应 = 开发井总数 |
| **跨专业** | 跨专业指标的匹配关系 | 高峰产量不应超过地面处理能力 |
| **版本变更** | 两版方案之间参数变了，应说明原因 | 建设周期从 24 改成 20 个月却没写原因 |
| **术语** | 同一事物应使用统一名称 | 正文混用「开发井总数」和「部署井数」 |
| **完整性** | 该有的章节/表格/状态不能缺 | 审查意见回复表漏填处理状态 |
| **产能 / 证据** | 产能关系、证据是否充分 | — |

### 4.4 「需人工复核」和「缺版本」是什么意思

- **「需人工复核」**：工具遇到自己**没有十足把握**的情况，宁可交给人判断，也**绝不擅自说「没问题」**。看到它，就代表「这里请你亲自确认一下」。
- **「存在缺版本的参数事实」**：当你只上传了**一版**方案时，工具无法判断某个参数「有没有相对上一版发生变更」，于是标为需人工复核。这是**正常且谨慎**的表现，不是报错。如果你想做版本对比，需要把两版方案都交给管理员配置进来。

### 4.5 如果没发现问题

会显示：「本次未发现规则可判定的问题。这并不代表方案完全正确，请仍由专家复核。」

请**务必认真对待这句话**——工具只查它会查的那几类问题，查不到的问题多得多。

## 五、专家复核：对每条问题做处理

现在每一张问题卡片的**底部**都有一排复核控件，你可以直接在网页上操作：

1. **专家结论**（下拉框）：选择「确认 / 驳回 / 已解决」。
   - **确认**：认可这是个真问题，需要整改。
   - **驳回**：判断这不是问题（比如工具误报）。
   - **已解决**：问题已经处理完毕。
2. **复核备注**（文本框）：写补充说明、依据、处理意见；如果你觉得工具给的标题/严重度需要调整，也在这里写清楚建议。
3. 点**「保存复核」**按钮。保存成功后，卡片顶部的「复核：」状态会立即变成你选的结论。

> 重要：你的处理**不会覆盖**工具原始的初审结果——工具原来的判断和你的复核结论、备注都会保留，便于追溯「AI 当初怎么判、人最后怎么定」。

## 六、导出报告

初审完成后，结果区顶部会出现**三个导出按钮**，点一下就会下载对应文件：

| 按钮 | 是什么 | 用途 |
|---|---|---|
| **导出 Excel（含问题位置）** | 表格版问题清单，每条含「问题位置」列 | 便于逐条核对、分派整改 |
| **导出 Word 报告** | 文档版初审报告，每条含「问题位置」 | 便于打印、归档、走流程 |
| **导出匿名包** | 去除了所有敏感信息的结果包 | 便于对外交流、做效果评估；**不含**厂商名、模型、密钥、原始文件、也不含可读位置 |

**「问题位置」**会直接告诉你每个问题对应原文的**章节、第几段、或第几张表的第几行第几列**（例如「附件A关键参数表 表格 第9行第2列」「开发部署方案 第7段」），你据此翻回原方案精确定位、修改。

所有导出文件都印有「**AI 初审结果，不是正式审查结论**」。

## 七、AI 复核设置（可选，进阶）

页面上有一块可折叠的**「AI 复核设置（可选）」**。默认使用**内置 Mock**（不联网、确定性的演示复核），不填任何东西也能正常用，多数人无需理会这一块。

如果你的单位配置了在线大模型网关，可以让 AI 做更强的复核：

1. 展开「AI 复核设置」。
2. **复核方式**选「在线 LLM（Anthropic 格式）」。
3. **Base URL**：填网关地址（默认已预填一个示例 `https://api.deepseek.com/anthropic`）。
4. **模型**：填模型名（如 `deepseek-chat`）。
5. **API Key**：填密钥。**密钥只保存在本机的安全凭据库里，不会写进文件、不会显示回来、不会进导出报告**；保存后输入框会清空，只显示「已配置密钥 ✓」。
6. 点**「保存配置」**，再点**「测试连接」**确认能连上。

之后再上传方案，AI 复核就会用这个在线模型。**若在线模型连不上或超时，工具会自动跳过 AI 那一步、保留规则检查结果，不会报错崩溃。**

> 重要：一旦用在线模型，方案正文会发送到你配置的网关。**本工具只允许上传已脱敏的方案**，请确保上传的文件确实已脱敏；涉密、未脱敏的文件严禁上传，也严禁用在线模型处理。

## 八、删除案例（清理资料）

出于资料安全，删除案例分两步，**防止误删**：

1. 先「**移入回收站**」；
2. 再「**二次确认永久删除**」。

只有完成第二步，文件才会被真正删掉。这个操作一般也由信息科同事协助。

## 九、常见问题（FAQ）

**Q1：为什么只能传 `.docx`？我的方案是 PDF / 老版 .doc 怎么办？**
当前版本**只支持文本型 Word 文档（.docx）**。PDF、扫描件、老格式 `.doc` 暂不支持。请先用 Word 把方案**另存为 `.docx`**再上传。如果传了不支持的格式，页面会明确提示，不会偷偷跳过。

**Q2：上传后提示「上传失败」或「DOCX 解析失败」怎么办？**
- 确认文件确实是 `.docx`（不是改了后缀名的 PDF）；
- 确认文件没有损坏、能正常用 Word 打开；
- 文件不要过大（上限 100MB、约 300 页）；
- 仍不行，请把文件和错误提示截图发给信息科同事。

**Q3：页面出现黄色提示「未加载规则库」怎么办？**
先点页面上方的**「加载 / 重新加载规则库」**按钮试一次——正常情况下工具会自动找到规则库并显示「已加载 12 条审查规则」。如果点了还是「未加载」，说明这台电脑上找不到示例规则包，请联系**系统管理员/信息科**（技术上是设置 `REVIEW_DEMO_ROOT` 指向规则包目录）。此时工具仍能做很浅的 AI 检查，但**没有跑完整的 12 条规则**。

**Q4：结果里有一堆「版本变更 / 缺版本」的提示，是不是报错了？**
不是。这是你只传了**一版**方案时的正常谨慎提示（见 4.4）。要做版本对比，需要管理员把两版方案一起配置进来。

**Q5：页面打不开 / 提示无法访问？**
说明后台服务没在运行。请联系信息科同事，让他们「启动服务」（运行程序）。记住这个网址只能在**装了程序的那台电脑**上打开。

**Q6：这些示例里的规则、数据是真的吗？**
当前随附的规则、术语、样例方案都是**虚构的演示数据**（标记为 `DEMO_ONLY`），**仅供试用和熟悉操作**，不构成任何正式审查依据。要用于实际业务，必须由专家建立**经确认的正式规则集**，绝不能把演示数据和正式规范混用。

## 十、红线与注意事项（务必遵守）

1. **这是初审，不是结论**——任何结果都必须经专家复核，不能直接作为正式审查依据或对外结论。
2. **不要过度依赖「没发现问题」**——工具查不到的问题远多于它能查的。
3. **涉密资料按规定处理**——虽然工具在本机运行、不外传，但上传、导出、留存的文件仍属于你单位的资料，请按保密规定管理；对外交流优先用**匿名包**。
4. **发现工具报错或明显误判**，请记录下来反馈给信息科/管理员，帮助持续改进。

## 十一、快速操作卡（打印贴在工位）

1. 浏览器打开 `http://127.0.0.1:8765`
2. 点「选择 DOCX」→ 选你的方案（必须 `.docx`）
3. 点「上传并开始」→ 等几秒
4. 看问题卡片：**红/橙优先**，「需人工复核」重点看
5. 需要报告 → 找信息科导出 Excel / Word
6. **牢记：AI 初审结果，不是正式审查结论**
## 当前使用边界

本工具只接受文本型 DOCX。PDF、OCR、RAG、知识图谱和复杂历史前端尚未实现。默认 AI 为不联网的 No-op Mock；在线 Anthropic 适配器必须显式配置，公共 URL 默认 HTTPS，私网/本机 URL 需要打开开关并先完成脱敏及组织审批。AI 初审结果不是正式审查结论，规则结果与 AI 结果都需要专家复核。支持的正式启动方式是 `python scripts/run_local.py`，仅监听 `127.0.0.1`；不要把第三方 uvicorn 启动参数当作公网安全部署方案。
